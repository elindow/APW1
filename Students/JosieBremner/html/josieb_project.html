Takeaways from: CSS Foundations

CSS Foundations (1/27/22): To make a structure, made from HTML of a webpage look good CSS is used. CSS is made of various rules. The rules are made of a selector and a semi-colon list with declarations that are made up of a property:value pair. Selectors refer to the HTML elements which CSS rules apply to. They are what is literally being selected for each rule. The universal selector selects elements of any type, the syntax for it is a asterisk (*). A type selector will select all the elements of the element type. The syntax for it is the name of the element if < div > then it is div. Class selectors select all elements with the given class. The syntax is a period and then without a space followed by case-sensitive value of the class attribute. ID selectors select an element with the given ID (that is another attribute placed on an HTML element). An ID cannot be repeated on a single page. Grouping Selectors are for when two groups of element share some of their style declarations. Chaining selectors work as a list without any seperation. Descendant combinator is represented in CSS by a single space between selectors. A combinator allows to combine multiple selectors in a different way than grouping or chaining them. They show a relationship between the selectors. A descendant combinator will only cause elements that match the last selector to be selected if they have an ansector that matches that previous selector. There are multipule CSS properties, here are some. Color and background-color, the color property sets and element's text color while the background-color sets the background color of an element. They use the word of an actual color but also accept HEX, RGB, and HSL values. Typography basics and text-align are in charge of the fonts of text. Image Height and width are in charge of controling the size of an image. A CSS declaration that is more specific will take over ones that are less specific. Selectors that contribute to specificity are ID selectors, Class selectors, and Type selectors.

Takeaways from: Flexbox

Introduction to Flexbox (2/24/22):Flexbox is a newish way to manipulate elements in CSS. It has been known to be revolutionary. It is used to arrange items into rows or columns where those items will either grow or shrink based on simple rules that you can define. It is a whole toolbox of properties that can be used to put things where needed. Some of the properties belong on the flex container and some go on the flex items. A flex container is any element that has "display: flex" on it. A flex item lives inside of a flex container. There are also elements that can be both.

Growing and Shrinking (3/4/22): This section teaches about what happens when you put "flex: 1" on flex items. The flex declaration is a shorthand for 3 properties that can be set on a flex item. These properties affect how flex items size themselves inside their container. Shorthand properties are CSS properities that let you set the values of multiple CSS properties at the same time. Shorthand properties help you write more concise stylesheets (which help you save time and energy). Flex is short for "flex-grow", "flex-shrink", and "flex-basis". Respctivly, the defult values are 0, 1, and 0%. "Flex-grow" expects a single number as its value. The is the flex-item's "growth value". Flex-shrink sets the "shrink factor" of the flex item. It only ends up being applied if the size of all flex items larger than their parent container. Due to the defult, all items will shrink evenly. If you don't want an item to shrink you can set the value to 0. Flex items do not necessarily respect your given values for width. Flex-basis simply setes the inital size of a flex-growing or flex-shrinking starts from that baseline size. Using auto as a flex-basis tells the item to check for a width declaration (width: 250px).

Axes (3/10/22): Flexbox can work either horizontally or vetically. The defult direction is horizontal, or (row). But there are ways to change the direction to vertical, or (column). The flexbox container should be though as having 2 axes: the main axis and the cross axis. The direction of these axes change the flex-direction. Usually, row puts the main axis horizontal and column puts the main axis vertical. When the flex-direction is changed to column, flex-basis refers to height instead of width. The defult behavior is flex-direction: row. This often works well without changing other details in the CSS becuase block-level elements defult to the height of their content.

Alignment (3/30/22): Flex is also very useful for arranging items that have a specific size. This section teaches how to align items inside a flex container both vertically and horizontally. By adding flex: 1 to .item makes each of the items grow to fill the avaliable space. justify-content aligns items across the main axis. If that is chnaged to center will change the items to center along the main axis. To change the placement of the items along the cross axis use align-items. For justify-content and align-item their behavior changes when you change the flex-direction of a flex-container. The defult behavior is the most common; this is justofy-content aligns items horizontallly and align-items aligns them vertically. Gap property is a useful feature of flex. When setting gap on a flex container it simply adds a specified space between flex items. Gap is a new property so it only mostly works on all modern browserss.
